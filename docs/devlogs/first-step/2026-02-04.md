## 04/02/2026 - Adicionar HTML antigo através dos templates

### Objetivo do dia
Melhorar a parte visual do início (url vazia) e reorganizar a estrutura dos arquivos de mídia.

### O que implementei
- ImageField no model MenuItem
```python
    image = models.ImageField(upload_to='menu_photos/%Y/', blank=False, null=False)
    #organização passa a ser separada por anos com '/%Y/'. melhor a longo prazo do que a organização anterior.
```
- Templates settings.py
```python
    TEMPLATES = [
        {
            'BACKEND': 'django.template.backends.django.DjangoTemplates',
            'DIRS': [BASE_DIR / "templates"], # <- adicionei o caminho para os templates
            'APP_DIRS': True,
            'OPTIONS': {
                'context_processors': [
                    'django.template.context_processors.request',
                    'django.contrib.auth.context_processors.auth',
                    'django.contrib.messages.context_processors.messages',
                ],
            },
        },
    ]
```
- Criei os diretórios templates/menu, static/menu/css, static/menu/js e static/menu/img e adicionei os arquivos do projeto de HTML, CSS e Javascript básicos que criei em Agosto de 2025. [Confira aqui o projeto original.](https://github.com/luccatrevisan/chewie-cookies-menu).

### Decisões
- **Por que escolhi criar as pastas no diretório base ao invés de criar as pastas dentro do aplicativo?**
  Fiz essa escolha por meu frontend ser único e simples e por ter somente um app no projeto todo. Eu poderia - e vou - criar outros apps, como app para login e autenticação, app para pagamentos etc. Porém, ainda ficaria organizado dentro do que essa organização propõe. Além de centralizar templates/static no diretório base evita duplicação e facilita manutenção.

### Desafios
1. Erro na hora de carregar o template
   - **Como resolvi:** Pesquisei na documentação oficial do django sobre templates e percebi que tinha esquecido a configuração em settings.py. Achava que o problema estava na configuração de views.py ou urls.py e fiquei sofrendo à toa. Começando a sentir na pele que ser desenvolvedor realmente é ficar meio perdido com os problemas mais simples.
