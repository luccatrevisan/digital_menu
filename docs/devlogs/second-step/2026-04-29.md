- API Fetch function
```javascript
async function apiFetch(){
    try{
        const token = await login("username", "password"); // note: these parameters are only placeholders.

        const response = await fetch("http://127.0.0.1:8000/api/item-by-category/", {
            method : "GET",
            headers : {
                "Authorization" : `Bearer ${token}`,
                "Content-Type" : "application/json"
            }
        });

        if (!response.ok){
            throw new Error("Could not fetch resource");
        }

        const data = await response.json();
        console.log(data);
    }
    catch(error){
        console.error(error);
    }
}
```
 
- Login function to get the token
```javascript
async function login(username, password){
    try{
        const response = await fetch("http://127.0.0.1:8000/api/token/", {
            method : "POST",
            headers: {
                "content-type" : "application/json"
            },
            body: JSON.stringify({
                username : username,
                password : password,
            })
        });

        const data = await response.json();
        return data.access;

    }
    catch(error){
        console.error(error)
    }
}
```

### Key Decisions
- **Why I chose AllowAny in MenuItemByCategoryViewSet:**
    I decided to expose the menu endpoint as public (read-only), allowing any user to view the menu without authentication. This avoids requiring credentials on the frontend and aligns with real-world applications where browsing products does not require login, while actions like ordering will require authentication.

- **Separation between templates and API logic:**
    Since the goal is to validate the backend logic and not build a production-grade frontend, I chose Django Templates with vanilla JS to keep the stack simple and avoid context switching. A decoupled frontend could be a future iteration but, this way, I can still use the API-oriented architecture.

### Challenges
1. Building the Fetch Logic with JWT Authentication
   - **How I solved it:** I learned how to access the API via the fetch() function, but I didn't know how to get the JWT token so that the JS logic could return the whole MenuItemByCategory endpoint. For that, I created another function (login()) just to get the token.
   - **What I learned:** I learned how JWT authentication works in practice on the frontend, including how to structure HTTP requests and its headers, body and parameters, attach tokens to Authorization headers, and handle authenticated vs public endpoints differently.
