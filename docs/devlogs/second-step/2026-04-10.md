### O que implementei
- Instalei a biblioteca do DRF simple JWT
```bash
pip install djangorestframework-simplejwt
```

```python
INSTALLED_APPS = [
    'rest_framework_simplejwt',
]
```

- Adicionei as classes de autenticação e de permissões
```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework_simplejwt.authentication.JWTAuthentication',
        'rest_framework.authentication.SessionAuthentication', # percebi mais tarde que Session Authentication não seria tão necessário, mas deixei no código por conta do admin
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
}
```

- Criei urls em /setup para gerenciamento dos tokens
```python
urlpatterns = [
    path('api/token/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
    path('api/token/refresh/', TokenRefreshView.as_view(), name='token_refresh')
]
```

- Adicionei classes de permissão e de autenticação nas views da API
```python
authentication_classes = [JWTAuthentication]
permission_classes = [IsAuthenticated]
```

- Implementei configurações do drf-spectacular para evitar problemas no Swagger 
```python
SPECTACULAR_SETTINGS = {
    "SECURITY": [{"BearerAuth": []}],

    "COMPONENTS": {
        "securitySchemes": { 
            "BearerAuth": {
                "type": "http",
                "scheme": "bearer",
                "bearerFormat": "JWT",
            }
        }
    }
}
```
### Decisões
- **Como fiz o backup do banco de dados:**
    Há algumas semanas, utilizei as fixtures para fazer a importação dos dados do SQLite3 para o PostgreSQL. Utilizei a mesma solução aqui, mas com algumas alterações, já que estava tendo alguns problemas com acentuação no banco. Já tinha criado um arquivo backup.json utilizando 'python manage.py dumpdata' mas, como estava com acentuação corrompida, fiz algumas trocas simples à mão e agora tenho o backup.json correto e completo. Para importar os dados, bastou usar o comando 'python manage.py loaddata backup.json'. Simples. A maior parte já estava resolvida.

- **Por que escolhi JWT ao invés do TokenAuthentication padrão do Django REST Framework:**
    Porque JWT não exige consultas ao banco de dados para validar o usuário. A autenticação é feita apenas verificando a assinatura do token enviado no header Authorization, o que melhora a escalabilidade e é mais adequado para APIs REST.

- **Implementação do BearerAuth para o Swagger:**
    Após proteger os endpoints com IsAuthenticated, o Swagger passou a exigir autenticação para testar as rotas. Configurei BearerAuth no drf-spectacular para permitir o envio do JWT no header Authorization, possibilitando testar endpoints autenticados diretamente pela interface do Swagger.
