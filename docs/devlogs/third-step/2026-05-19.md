## 19/05/2026 - [ORDERS] Implement atomic order creation logic
### Goal
Ensure that order creation is safe and consistent by wrapping all operations in atomic transactions, preventing partial data persistence in case of failure.

- apps/orders/services/create_order.py: the main part of the day
```python
def create_order(items_data): 
    try:
        with transaction.atomic():
            order = Order.objects.create()

            for item_data in items_data:
                menu_item_id = item_data.get("menu_item_id")
                quantity = item_data.get("quantity")

                # validates menu item existence
                try:
                    menu_item = MenuItem.objects.get(id=menu_item_id)

                except MenuItem.DoesNotExist:
                    raise ValidationError(
                        f"Menu item with id {menu_item_id} does not exist."
                    )

                # validates stock existence
                try:
                    stock = Stock.objects.select_for_update().get(menu_item=menu_item)

                except Stock.DoesNotExist:
                    raise ValidationError(
                        f"Stock for '{menu_item.name}' does not exist."
                    )

                # validates stock quantity
                if stock.quantity is not None and quantity > stock.quantity:
                    raise ValidationError(
                        f"Insufficient stock for '{menu_item.name}'. "
                        f"Available: {stock.quantity}"
                    )

                # creates order item
                OrderItem.objects.create(
                    order=order,
                    menu_item=menu_item,
                    quantity=quantity
                )

                # decreases stock
                stock.quantity -= quantity
                stock.save(update_fields=["quantity"])

            # updates total order price
            order.update_total_price()

            return order

    except ValidationError as e:
        raise ValidationError(
            {"order_error": e.message}
        )

    except Exception as e:
        raise ValidationError(
            f"Unexpected error while creating order: {str(e)}"
    )
```
 
### Key Decisions
- **Why use transaction.atomic():**
    basically, to avoid doing unnecessary work. to create an order, i have to work through both Order and OrderItem models. if it fails, the objects would be created without transaction.atomic() and would consume space unnecessarily.

- **What should I use select_for_update for:**
    to avoid the classic concurrency problem of two users placing an order at the same time. if an user A starts their order but an user B finishes it first, considering a stock of only one item, B gets it because he locks the stock.quantity row on the database.
