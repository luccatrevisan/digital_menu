## 08/05/2026 - built the register logic
### Goal
Connect register form to the brand new register API endpoint, enabling new users to create an account and authenticate automatically.

### Implemented
- Register function:
```javascript
async function register(url, username, email, phoneNumber, password){
    try{
        const response = await fetch(url, {
            method : "POST",
            headers : {
                "Content-Type" : "application/json",
                "X-CSRFToken" : getCSRFToken(),
            },
            body : JSON.stringify({
                username : username,
                email : email,
                phone_number : phoneNumber,
                password : password
            })
        });

        if (!response.ok){
            const errorData = await response.json();
            if (errorData.username){
                throw new Error("user_registered");
            }

            if (errorData.email){
                throw new Error("email_registered");
            }

            if (errorData.password){
                throw new Error("weak_password");
            }

            throw new Error("Erro ao cadastrar usuário");
        }

        return;

    } catch(error){
        throw error;
    }
}
```

### Key Decisions
- **Where should I put the password validation?**
    First, I put the if statement between password and confirmPassword inside the register function. But I realized that the function would be doing too much.

    My second thought was to put it on the form level because it would be faster. If this validation were happening after the submission, would be a waste of the user's time and the data submitted all the way to the server would be invalid.

- **Why I added 'export' on the login function:**
    To build good experience for the user after successful registration, the frontend automatically authenticates the user using the existing login() function. This avoids duplicating token generation logic and keeps authentication concerns separated from account creation. 

- **Should I use CSRF token treatment for register too?**
    Yes. Without CSRF protection, a malicious website could submit a forged POST request to /api/register/ using the victim's session. Since Django validates the CSRF token on every unsafe method, including it ensures the request genuinely originates from our form.
