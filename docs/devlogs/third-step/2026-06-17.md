## 17/06/2026 - [ORDERS] Create order checkout endpoint
### Goal
Provide an API endpoint capable of receiving cart data and creating orders

### Implemented
- Create Order View 
```python
class OrdersView(generics.CreateAPIView):
    authentication_classes = [JWTAuthentication]
    permission_classes = [IsAuthenticated]
    serializer_class = OrderCreateSerializer
```
 
- Create a post() function inside the View
```python
def post(self, request):
    with transaction.atomic():
        serializer = self.get_serializer(data=request.data)
        serializer.is_valid(raise_exception=True)

        order = Order.objects.create(
            user = request.user
        )

        for item_data in serializer.validated_data["items"]:
            menu_item = MenuItem.objects.get(pk=item_data["menu_item_id"])

            OrderItem.objects.create(
                order=order,
                menu_item=menu_item,
                quantity=item_data["quantity"]
            )

        return Response({"order_id" : order.id}, status=status.HTTP_201_CREATED)
```

### Key Decisions
- **Why I chose JWT authentication at the API level instead of relying only on frontend checks:**
Although the frontend hides the checkout flow from unauthenticated users, the API itself must remain protected. By requiring JWT authentication with `IsAuthenticated`, the backend becomes the source of truth and prevents unauthorized requests regardless of frontend behavior.

- **Why I chose transaction.atomic():**
Order creation involves multiple database operations. Wrapping the process in a database transaction ensures that either the order and all its items are created successfully, or none of them are persisted. This prevents partial orders from being stored in case of failures during checkout.

### Challenges
1. I had some problems trying to check if the user was logged in for the order
   - **How I solved it:** Initially, the checkout endpoint was returning authentication-related errors even when the frontend appeared to be sending a valid JWT token. I inspected the browser's Network tab, verified the Authorization header, decoded the JWT payload, and traced the request through the backend. During the investigation, I also discovered unrelated issues such as missing imports and outdated local tokens from previous development sessions.

   - **What I learned:** Authentication problems should be debugged systematically by validating each layer independently: token generation, browser storage, request headers, DRF authentication classes, and endpoint permissions. The browser Network tab and Django server logs were the most useful tools during the investigation.
