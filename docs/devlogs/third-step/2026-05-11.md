### Goal
Ensure the full integration between Django templates, JavaScript, and the API works as expected. Today was a simpler day, without any challenges, more focused on fixing bugs.

### Implemented
- Changed the phone_number attribute on the CustomUser model:
```python
class CustomUser(AbstractUser):
    email = models.EmailField(unique=True)
    phone_number = models.CharField(max_length=20, unique=True)  # added unique=True to avoid accounts registering with the same phone
```
 
- Fixed the error treatment on register.js:
```javascript
if (!response.ok){
    const errorData = await response.json();
    if (errorData.username){
        throw new Error(errorData.username[0]);
    } else if (errorData.email){
        throw new Error(errorData.email[0]);
    } else if (errorData.password){
        throw new Error(errorData.password[0]);
    } else if (errorData.phone_number) { // Added treatment for phone number error.
        throw new Error("Usuário com este número de celular já existe.") // For this, I preferred a custom message because Django does not translate "phone number" to pt-br correctly
    } else {
        throw new Error("Erro ao cadastrar usuário");
    }
}
```
 
- Kept the catch block on the register form event listener simpler, using the Django messages:
```javascript
catch(error){
    const errorContainer = document.getElementById("register-error");
    errorContainer.textContent = error.message; // shows the messages from the request and, for the phone_number error, as seen above, the custom message.
}
```
 
### Key Decisions
- **Why I changed the phone_number attribute:**
    I checked if I could register an user with an already registered phone number and it worked. I changed the attribute phone_number to unique=True on the CustomUser model to be able to build a logic that avoids this.
    
- **Why I changed the error treatment block on the register() function:**
    I noticed that the login page was working properly, but the register page was returning confusing error messages. If I used an username that was NOT used before, the message of "username already on use" would appear. That was happening because of the error treatment block on the register() function. I changed it to return the messages created by the Django REST Framework to keep the logic simple but to also fix my previous logic. By following the DRF structure, I can also show the user specific problems instead of one customized message.
