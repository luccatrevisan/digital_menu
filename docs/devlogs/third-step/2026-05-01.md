- views.py on the apps.users folder
```python
from django.shortcuts import render

# Simple render views for initial template validation (will evolve with API integration)
def login(request):
    return render(request, "users/login.html")

def register(request):
    return render(request, "users/register.html")
```

- urls.py on the apps.users folder and the setup folder
```python
# apps.users.urls:
from django.urls import path
from apps.users.views import login, register

urlpatterns = [
    path("login/", login, name="login"),
    path("register/", register, name="register")
]

# setup.urls
urlpatterns = [
    path('', include("apps.users.urls")), # included the urlpatterns from users
]
```
 
### Key Decisions
- **Template organization by domain:**
    I reorganized the template structure to separate files by app (users, menu) and introduced a shared folder for reusable layouts (base.html). This improves scalability and keeps responsibilities well defined as the project grows.

- **Why I removed the static() function from apps.menu.urls:**
    Mainly, because I was already using the function on setup.urls and used the include() function to get the urlpatterns arrays from apps.menu and apps.users. It would be redundant to have the same functionality on both files and the responsability would be divided between them. By keeping static() only on setup.urls, the project can be clearer.

- **Validation of template rendering flow:**
    I focused on validating the full rendering pipeline (URL -> view -> template) to ensure the frontend structure is correctly integrated before adding dynamic API behavior.
